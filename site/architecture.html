Trust & Architecture

Architecture & Trust Model

Stele's correctness guarantee rests on a single small trusted core. Every other layer is explicitly untrusted and advisory.

Trust Layers

Trusted core Kernel
142-line pure syntactic matcher. First-order rule matching only. No semantic reasoning. De Bruijn criterion. Stdlib only. The sole source of proof validity verdicts.
stele/kernel.py
Trusted Parser
Hand-written recursive descent tokenizer and parser. No external parser library. Stdlib only. Produces AST consumed by kernel.
stele/parser.py
Untrusted Diagnostics
Structural multi-pass analysis: unused assumptions, missing hypotheses, scope errors, undefined symbols. Advisory only — no kernel import.
stele/diagnostics.py
Untrusted Proof-state hints
Rule suggestion hints based on current proof state. All hints carry trusted=False — never used by kernel.
stele/proofstate.py
Experimental Certificates
Kernel-gated certificate emission. Minicheck provides an independent Python re-verification path — same process, different code path.
stele/certificate.py · stele/minicheck.py
Optional ML Baseline
Multinomial Naive Bayes classifier. Synthetic corpus. Completely isolated from trust core. No effect on kernel correctness.
stele_ml/
