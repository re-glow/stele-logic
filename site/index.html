What Stele Does

A proof checker with semantic diagnostics

Stele is a proof checker, not a theorem prover. You supply each inference step; the kernel decides if it is valid under the declared logic. Multiple logics can be compared side-by-side.

Tutorial

5-minute walkthrough

Learn the proof format and main features step by step. Skip to Studio →

1 — The proof format

Stele proofs follow natural-deduction structure with five keywords:

  • theorem NAME: — declares the theorem
  • assume LABEL: FORMULA — introduces a global premise
  • suppose LABEL: FORMULA … end — opens a subproof block
  • have LABEL: FORMULA by RULE REFS… — derives a step using a named rule
  • conclude FORMULA by LABEL — states the final conclusion

The trusted kernel checks every have step against the declared logic's inference rules. The example on the right proves P → P (implication identity) intuitionistically. 

theorem imp_self:
  suppose h1: P
    have h2: P by copy h1
  have h3: P -> P by imp_intro h1 h2
  conclude P -> P by h3

2 — Catch an error

The kernel rejects each step that does not satisfy its rule. Here mp (modus ponens) requires:

  • a premise of form A → B — h1 is P → Q ✓
  • a premise matching the antecedent A — h2 is R, not P ✗

The Diagnose panel classifies the error: InvalidTransition, UndefinedSymbol, UnusedAssumption, and more — each with a line number. 

theorem bad_mp:
  assume h1: P -> Q
  assume h2: R
  have h3: Q by mp h1 h2
  conclude Q by h3

3 — Dependency graph

The Graph panel extracts proof structure as a dependency graph. Each node is a step; directed edges show which steps cite which hypotheses.

Output is DOT notation. Paste it into Graphviz Online ↗ to visualise the proof tree. Useful for spotting unused assumptions and isolated subproofs. 

theorem chain:
  assume h1: P -> Q
  assume h2: Q -> R
  assume h3: P
  have h4: Q by mp h1 h3
  have h5: R by mp h2 h4
  conclude R by h5

4 — Classical vs intuitionistic

The same proof can be accepted or rejected depending on the active logic. Double negation elimination (dne) is a rule in classical logic but not in intuitionistic logic.

The kernel never changes — only the active rule set does. This is proof relativity at the rule-availability level. Switch the logic selector in the Studio to compare. 

theorem dne_consequent:
  assume h1: not not P
  have h2: P by dne h1
  conclude P by h2

5 — Semantic tools (⊢ vs ⊨)

Stele separates syntactic provability (⊢, proof checking) from semantic validity (⊨). The Semantics panel provides two independent tools:

  • Rule Soundness — checks whether each inference rule preserves designated values in K3 (strong Kleene), LP (Logic of Paradox), or Boolean matrices.
  • World Lattice — shows a formula's status (provable, refutable, or independent) across worlds that differ in one axiom.

These tools operate independently of proof checking — no proof text required.
