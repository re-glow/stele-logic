What Stele Does

A proof checker with semantic diagnostics

Stele is a proof checker, not a theorem prover. You supply each inference step; the kernel decides if it is valid under the declared logic. Multiple logics can be compared side-by-side.

⊢

Proof Verification

Write proofs in a readable natural-deduction language. The trusted kernel verifies every step against the declared logic (intuitionistic, classical, or a custom rule set).

⚠

Structural Diagnostics

Multi-pass analysis flags undefined symbols, missing hypotheses, invalid transitions, and unused assumptions with exact line numbers.

⬡

Dependency Graph

Extract the proof's step-by-step dependency graph. Detect cycles, isolated steps, and unused assumptions. Export in DOT notation.

⊨

Semantic Tools

Check rule soundness against multi-valued matrices (K3, LP, Boolean). Explore world lattices and proposition-level independence patterns.

Proof relativity The same proof accepted by classical logic may be rejected by intuitionistic logic — switch the logic selector and see instantly. The kernel never changes; only the rule set does.

Valid · Intuitionistic

        
Hypothetical Syllogism

        
From A→B and B→C derive A→C. Valid in all logics.

        
theorem imp_chain:
  assume h1: A -> B
  assume h2: B -> C
  suppose h3: A
    have h4: B by mp h1 h3
    have h5: C by mp h2 h4
  have h6: A -> C by imp_intro h3 h5
  conclude A -> C by h6

        
      


      

        
Classical only

        
Double Negation Elimination

        
¬¬P → P. Accepted classically, rejected intuitionistically.

        
theorem dne_consequent:
  assume h1: not not P
  have h2: P by dne h1
  conclude P by h2

        
      


      

        
Invalid

        
Type Mismatch

        
mp requires a matching antecedent — this one is wrong. Diagnostics show the exact error.

        
theorem bad_mp:
  assume h1: P -> Q
  assume h2: R
  have h3: Q by mp h1 h2
  conclude Q by h3

        
      


      

        
Valid · Intuitionistic

        
Negation Introduction

        
Prove ¬(P ∧ ¬P) by assuming it and deriving ⊥.

        
theorem neg_intro_demo:
  suppose h1: P and not P
    have h2: P by and_elim_left h1
    have h3: not P by and_elim_right h1
    have h4: false by neg_elim h2 h3
  have h5: not (P and not P) by neg_intro h1 h4
  conclude not (P and not P) by h5

        
      


      

        
Classical only

        
Law of Excluded Middle

        
P ∨ ¬P derivable classically (lem rule). Rejected intuitionistically.

        
theorem excluded_middle:
  have h1: P or not P by lem
  conclude P or not P by h1

        
      


      

        
Valid · Both

        
Conjunction Introduction

        
From P and Q, construct P ∧ Q, then extract both components.

        
theorem and_demo:
  assume h1: P
  assume h2: Q
  have h3: P and Q by and_intro h1 h2
  have h4: P by and_elim_left h3
  have h5: Q by and_elim_right h3
  conclude Q by h5
