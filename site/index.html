What Stele Does

A proof checker with semantic tools

Proof checker, not theorem prover. You supply each inference step; the kernel decides if it is valid under the declared logic.

⊢

Proof Verification

Stable

The trusted kernel verifies every step against the declared logic — intuitionistic, classical, or a custom rule set.

⚑

Structural Diagnostics

Stable

Flags undefined symbols, missing hypotheses, and invalid transitions with exact line numbers.

◈

Dependency Graph

Stable

Extracts the proof's step-by-step dependency graph, detects cycles, and exports DOT notation.

⬡

Semantic Tools

Stable Kripke — Experimental

Check rule soundness against K3/LP/Boolean matrices; find bounded Kripke countermodels for intuitionistic logic.

Studio

The Stele Studio runs entirely via Pyodide/WASM — no installation, no backend, browser-local. Paste a proof, pick a logic, and get an instant kernel verdict.

Multi-Logic

The logic is a parameter, not a fixed assumption.

Some rules that feel obvious in classical logic fail under other semantics. The same proof text, a different logic selector, a different verdict.

classical_prop Accepts 
theorem dne:
  assume h1: ¬¬P
  have h2: P by dne h1
  conclude P by h2

✓ Valid

dne (double-negation elimination) is a classical rule — available in classical_prop.

intuitionistic_prop Rejects 
theorem dne:
  assume h1: ¬¬P
  have h2: P by dne h1
  conclude P by h2

✗ Rejected

dne is not in the intuitionistic rule set. The kernel rejects this step.

Try ¬¬P → P in the Kripke checker. Classical logic accepts it. Intuitionistic semantics can find a model where it fails. Open Studio →

Tutorial

Learn Stele in 5 minutes

Skip to Studio →

Write your first proof

A Stele proof names hypotheses and cites a rule for each step. The kernel checks every citation.

theorem imp_self:
  suppose h1: P
    have h2: P by copy h1
  have h3: P -> P by imp_intro h1 h2
  conclude P -> P by h3

Verify with the kernel

Select a logic and click Verify. The kernel checks each rule citation against that logic's schema — intuitionistic or classical.

-- select: intuitionistic_prop
-- click: Verify
-- result: ✓ Valid · 3 steps

Catch an error

Stele's diagnostics locate the exact step with the wrong rule citation — undefined symbol, wrong premise type, missing hypothesis.

theorem bad_mp:
  assume h1: P -> Q
  assume h2: R          -- wrong type!
  have h3: Q by mp h1 h2
  conclude Q by h3

Inspect the dependency graph

The graph view extracts the proof's step dependencies, detects cycles, and exports DOT notation for visualization tools.

-- theorem node
-- → suppose/assume nodes
-- → have nodes → conclude node
-- each edge: "cited by" relation

Compare logics

The same proof can be valid under classical logic but rejected by intuitionistic logic. Switch the logic selector to see the verdict change.

theorem dne:
  assume h1: not not P
  have h2: P by dne h1
  conclude P by h2
-- classical_prop: ✓ Valid
-- intuitionistic_prop: ✗ Rejected
