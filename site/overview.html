Identity

What Stele Is

Stele is a proof checker and formal verification framework for natural-deduction proofs. The user writes explicit proof steps in a declarative surface language; the trusted kernel (stele/kernel.py) checks whether each inference step is valid under the declared logic. Stele verifies proofs — it does not search for them.

Rules are data (named RuleSchema objects), not hard-wired kernel branches. This means new logics can be added by supplying a rule set, without touching the trusted core. The same proof that passes under classical logic is rejected under intuitionistic logic, exactly as the theory predicts.

The trusted core follows the de Bruijn criterion: only kernel.py is in the trust boundary. Diagnostics, semantic tools, and proof-state hints are labelled "untrusted" and cannot affect the kernel's validity verdict.

Scope

What Is Not Claimed

These are deliberate scope limitations, not future bugs. Stele is designed to be a small, auditable proof checker — not a production theorem prover.

No proof search
Stele verifies proofs the user writes. It does not search for proofs, generate proof strategies, or fill in missing steps. Every step must be explicitly stated and named.
No machine-checked metatheory
Metatheory claims (subject reduction, normalization, consistency) are supported by regression tests and proof sketches. Tests are evidence, not machine-verified proof.
Propositional scope only
The proof-script language handles propositional logic only. First-order quantifiers (∀, ∃) exist in the proof-term layer (stele.core) but are not yet exposed at the surface level.
No "fully offline" mode
The browser Studio requires an internet connection to load Pyodide from CDN (~8 MB, cached after first visit). Fully offline mode is listed as future work.

Metatheory Status

Important — status of theoretical claims

Stele's proof-term core makes seven metatheory claims documented in docs/metatheory.md ↗. These include subject reduction, normalization, and consistency. All claims are supported by regression and property-based tests, plus written proof sketches. None is machine-checked by a proof assistant (Lean, Coq, Agda). The tests catch implementation bugs; they do not constitute mathematical proofs.
